Socket API

Mosaic exposes a JSON-RPC 2.0 API over a Unix domain socket. The default socket path is /tmp/mosaic.sock (tagged debug builds use /tmp/mosaic-<tag>.sock).

Protocol

The API follows the JSON-RPC 2.0 specification. Messages are newline-delimited JSON. Each line is one complete JSON-RPC message.

Request Format

{
    "jsonrpc": "2.0",
    "id":      "unique-request-id",
    "method":  "namespace.command",
    "params":  { "key": "value" }
}

The id field can be a string or integer. The params field is an object or array.

Success Response

{
    "jsonrpc": "2.0",
    "id":      "unique-request-id",
    "result":  { ... }
}

Error Response

{
    "jsonrpc": "2.0",
    "id":      "unique-request-id",
    "error":   {
        "code":    -32601,
        "message": "Method not found"
    }
}

Standard Error Codes

CodeMeaning
-32700Parse error -- invalid JSON
-32600Invalid request -- missing or wrong jsonrpc version
-32601Method not found
-32602Invalid params
-32603Internal error
-32000Access denied
-32001Application-level error (not found, invalid state)

Notifications (No ID)

Messages without an id field are notifications (fire-and-forget). The server processes them but does not send a response. Server-push events to subscribed clients also use the notification format. 

{
    "jsonrpc": "2.0",
    "method":  "process.exit",
    "params":  { "pane_id": "...", "exit_code": 1 }
}

Event Subscriptions

Clients can subscribe to server-push events via the EventBus. Subscribed events are delivered as JSON-RPC notifications (no id) pushed to the client over the same socket connection.

Available Events

EventDescription
process.exitA terminal process exited (includes exit code)
output.matchTerminal output matched a subscribed pattern
port.changeListening ports changed (detected by port scanner)
shell.idleShell has been idle beyond configured duration
workspace.changeWorkspace created, deleted, selected, or modified

Access Control

The socket server supports five access control modes, configured in Mosaic settings:

ModeBehavior
offAll connections rejected
localOnlyOnly child processes of Mosaic (PID ancestry verification)
automationAny local connection accepted, marked as automation context
passwordChallenge-response authentication (constant-time comparison)
allowAllAny local connection accepted without checks

Connection Example (Python)

import socket, json

sock = socket.socket(socket.AF_UNIX, socket.SOCK_STREAM)
sock.connect("/tmp/mosaic.sock")

request = {
    "jsonrpc": "2.0",
    "id": 1,
    "method": "workspace.list",
    "params": {}
}

sock.sendall((json.dumps(request) + "\n").encode())
response = sock.recv(8192).decode()
data = json.loads(response)

print(json.dumps(data, indent=2))
sock.close()
