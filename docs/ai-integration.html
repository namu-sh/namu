AI Integration

NamuAI is a natural-language control plane. It is not an agent bridge and not an MCP wrapper. Given a request like “split the pane and run npm test”, NamuAI gathers context, calls an LLM provider, maps tool calls to socket commands, safety-checks them, executes them, and returns a result.

Provider Model

Namu supports swappable LLM providers through a common LLMProvider abstraction.

ProviderImplementation
ClaudeNamuKit/AI/Providers/ClaudeProvider.swift
OpenAINamuKit/AI/Providers/OpenAIProvider.swift
GeminiNamuKit/AI/Providers/GeminiProvider.swift
CustomNamuKit/AI/Providers/CustomProvider.swift

Local Entry Points

namu ai message --content "create a new workspace called API"
namu ai status
namu ai history --limit 10

The current checkout ships local AI usage through the app UI and the CLI above.

Execution Model

User message
  --> ConversationManager
    --> ContextCollector snapshot
      --> LLM provider.complete(...)
        --> tool_use calls
          --> CommandMapper
            --> CommandSafety
              --> JSON-RPC handler
                --> formatted result

AI execution is async by default. The model can emit multiple tool calls in a loop, each of which is fed back into the conversation before the final answer is produced.

Command Safety

Every AI-issued command passes through CommandSafety:

  • safe — read-only commands like list, status, and screen reads
  • normal — structural changes like create, split, focus, resize
  • dangerous — input injection such as send-keys, send-text, execute-js

Destructive shell patterns always require confirmation. External callers should not bypass this layer.

Alerts and Channels

Namu includes a rule-based AlertEngine with 4 trigger types. This is separate from the LLM layer and does not require AI to function. 

EventBus event
  --> AlertEngine (4 trigger types)
    --> AlertRouter
      --> Slack / Telegram / Discord / Webhook

Alert Channels

  • Slack Channel — Native integration with credential security
  • Telegram Channel — Outbound delivery to Telegram bots
  • Discord Channel — Discord webhook support
  • Webhook Channel — Generic webhook delivery with custom headers and payload formatting

Alert Trigger Types

The AlertEngine supports 4 configurable trigger types for rule-based alert routing based on EventBus events (process.exit, output.match, port.change, shell.idle, workspace.change).

Claude Hook Integration

Namu supports 10 Claude hook event types for deep IDE integration. Install hooks via namu claude-hook commands:

  • claude-hook session-start — Triggered when a Claude session begins
  • claude-hook prompt-submit — Triggered when a prompt is submitted
  • claude-hook stop — Triggered when a Claude session stops

These hooks allow external tools to coordinate with Namu's terminal context.

Codex Integration

Namu ships Codex hook support for IDE-level automation: 

namu codex install-hooks        # Install into ~/.codex/hooks.json
namu codex uninstall-hooks      # Remove hooks
namu codex-hook session-start   # Handle session start events
namu codex-hook prompt-submit   # Handle prompt submit events

Remote Access Today

Remote forwarding in the active codebase is handled by the authenticated TCP RelayServer (HMAC-SHA256) plus helper tooling in daemon/remote/. Use namu system relay_status to inspect whether relay support is available and running.

Historical Gateway Note

Some older repo-local docs mention a standalone gateway/WebSocket bridge for remote AI or Telegram command ingress. That module is not part of the active checkout and should be treated as historical design material, not shipped functionality.

Error Handling

FailureBehavior
LLM API unavailableNamuAI returns a user-facing temporary-unavailable message.
Malformed tool callThe malformed call is skipped and the model is prompted to continue safely.
Socket command errorThe JSON-RPC error is formatted back into the AI response.
Safety rejectionThe command is blocked and the rejection reason is surfaced to the user.
Alert delivery failureThe channel error is logged without crashing the app.
