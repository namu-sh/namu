AI Integration

NamuAI is a natural-language control plane. It is not an agent bridge and not an MCP wrapper. Given a request like "split the pane and run npm test", NamuAI gathers context, calls an LLM provider, maps tool calls to socket commands, safety-checks them, executes them, and returns a result.

Provider Model

Namu supports swappable LLM providers through a common LLMProvider abstraction.

ProviderImplementation
ClaudeNamuKit/AI/Providers/ClaudeProvider.swift
OpenAINamuKit/AI/Providers/OpenAIProvider.swift
GeminiNamuKit/AI/Providers/GeminiProvider.swift
CustomNamuKit/AI/Providers/CustomProvider.swift

API keys are stored per-provider in the system Keychain. Switch providers in Settings → AI.

Local Entry Points

namu ai message --content "create a new workspace called API"
namu ai message --content "split the current pane and run pytest"
namu ai status
namu ai history --limit 10

Execution Model

User message
  --> ConversationManager
    --> ContextCollector snapshot
      --> LLM provider.complete(...)
        --> tool_use calls
          --> CommandMapper
            --> CommandSafety
              --> JSON-RPC handler
                --> formatted result

AI execution is async. The model can emit multiple tool calls in a loop, each fed back into the conversation before the final answer is produced.

Command Safety

Every AI-issued command passes through CommandSafety:

  • safe — read-only commands: list, status, screen reads
  • normal — structural changes: create, split, focus, resize
  • dangerous — input injection: send-keys, send-text, execute-js

Destructive shell patterns always require confirmation. External callers must not bypass this layer.

Claude Code Hook Integration

Namu supports 10 Claude hook event types for deep integration with Claude Code sessions. The integration uses a wrapper script that intercepts Claude Code invocations and injects session tracking.

Hook Event Types

  • SessionStart — Triggered when a Claude session begins
  • UserPromptSubmit — Triggered when a prompt is submitted
  • PreToolUse — Triggered before a tool call executes
  • Notification — Triggered on Claude notification events
  • Stop — Triggered when generation stops
  • SessionEnd — Triggered when the session ends
  • 4 additional hook types for fine-grained lifecycle events

What the Hooks Do

  • Track agent PID per surface (set_agent_pid claude_code <pid>)
  • Update sidebar status pill for Claude Code activity
  • Send targeted notifications to the workspace surface
  • Store session state at ~/.namu/claude-hook-sessions.json (7-day TTL)

Install

namu claude-teams          # install for current session
namu claude-hook session-start
namu claude-hook prompt-submit
namu claude-hook stop

Disable

export NAMU_CLAUDE_HOOKS_DISABLED=1

Codex Hook Integration

Similar hook support for OpenAI Codex with session-start, prompt-submit, and stop events. 

namu codex install-hooks
namu codex uninstall-hooks
namu codex-hook session-start
namu codex-hook prompt-submit

Agent PID Tracking

Namu tracks running agent processes per surface. This powers status display, process monitoring, and cleanup on workspace close. 

# Via socket API directly
{"method":"system.set_agent_pid","params":{"agent":"claude_code","pid":12345}}
{"method":"system.clear_agent_pid","params":{"agent":"claude_code"}}

Per-Workspace Port Allocation

Each workspace gets a dedicated port range, injected as environment variables into every terminal surface. This lets multiple agent sessions run servers without port conflicts. 

NAMU_PORT=3000
NAMU_PORT_END=3099
NAMU_PORT_RANGE=3000-3099

Alerts and Channels

Namu includes a rule-based AlertEngine with 4 trigger types. This is separate from the LLM layer and does not require AI to function. 

EventBus event (process.exit | output.match | port.change | shell.idle)
  --> AlertEngine (4 trigger types)
    --> AlertRouter
      --> Slack / Telegram / Discord / Webhook

Alert Channels

  • Slack — Native integration with credential security
  • Telegram — Outbound delivery to Telegram bots
  • Discord — Discord webhook support
  • Webhook — Generic delivery with custom headers and payload formatting

Configure channels in Settings → Alerts. Credentials are stored in the Keychain.
