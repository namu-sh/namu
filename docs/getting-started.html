Getting Started

Prerequisites

  • macOS 14.0 or newer
  • Xcode 16.0 or newer
  • zig 0.15.2
  • xcodegen

Clone and Setup

git clone --recursive https://github.com/omxyz/namu.git
cd namu
./Scripts/setup.sh
xcodegen generate
open Namu.xcodeproj

Scripts/setup.sh initializes the Ghostty dependency and builds the GhosttyKit.xcframework needed for the full app.

Build

xcodebuild -scheme Namu -configuration Debug build

Run Tests

xcodebuild -scheme Namu -configuration Debug test

xcodebuild -scheme Namu -configuration Debug \
  -only-testing:NamuTests/WorkspaceTests test

Repository Layout

namu/
  NamuKit/         Core logic: domain, terminal, IPC, AI, alerting, services
  NamuUI/          SwiftUI + AppKit app and views
  CLI/             `namu` CLI and hook integrations
  daemon/remote/   Remote relay helper/bootstrap
  Resources/       Info.plist, shell integration, bundled CLI, skills
  Tests/           Unit tests, UI tests, Python integration tests
  project.yml      XcodeGen project definition

First Run Checks

  1. Launch the Namu scheme from Xcode.
  2. Confirm a terminal workspace opens successfully.
  3. Verify the Unix socket exists at /tmp/namu.sock.
  4. From another terminal, run namu system ping.
  5. Optionally inspect available methods with namu system capabilities --json.

Optional Remote Relay

The app also includes an authenticated TCP relay (RelayServer) plus a helper under daemon/remote/. This is the active remote-access path in the current checkout.

Notes

  • Some older documentation still references a standalone gateway module. Treat those references as historical.
  • Current alert delivery is outbound only via configured channel credentials.
  • Current AI workflows are available locally through the app UI and namu ai * commands.
