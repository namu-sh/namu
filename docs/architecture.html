Architecture

Module Overview

Namu is organized around four main code areas with clear boundaries:

AreaDirectoryPurpose
NamuKit NamuKit/ Core logic. Domain models, Ghostty FFI, IPC, relay, services, AI, alerting. No SwiftUI imports.
NamuUI NamuUI/ SwiftUI and AppKit views. App entry point, workspace rendering, browser views, settings, AI chat, notifications, update UI.
CLI CLI/ The namu command-line client plus tmux compatibility, Claude hooks, Codex hooks, and helper entry points.
Remote Helper daemon/remote/ Remote relay helper/bootstrap for forwarded command access over an authenticated TCP relay.

NamuKit Internals

SubdirectoryResponsibility
Domain/Core value types such as Workspace, Panel, PaneTree, session snapshots, and workspace placement settings.
Terminal/Ghostty bridge/config/keyboard wrappers, TerminalSurfaceRegistry (pointer safety), PortalHostLease system, session lifecycle, shell integration, copy mode, image transfer (Kitty graphics + SCP).
IPC/SocketServer, RelayServer (TCP with HMAC-SHA256 auth), command registry/dispatcher, access control, event bus, command namespaces, and middleware.
Services/Workspace/panel managers, persistence, notifications, layout, appearance, analytics, PortScanner (batched ps+lsof), and app coordination.
AI/NamuAI, provider abstraction, context collection, command mapping, conversation state, command safety, and alert engine with 4 trigger types.
Alerting/Outbound channel delivery for Slack, Telegram, Discord, and generic webhook alerts with credential security.
Browser/BrowserProfileStore, BrowserHistoryStore, browser models, MarkdownPanel with file watching, helpers for 84+ browser commands.
Debug/Keystroke latency profiling (TypingTiming, always compiled with 34 instrumentation points).
Scripting/AppleScript support (4 classes, 11 commands via SDEF).
Config/Project configuration, directory trust, and command execution.

Key Design Decisions

Clean Rewrite

Namu references proven ideas from established projects — Ghostty embedding, socket namespaces, layout persistence, portal-style UI coordination — but it is a fresh codebase. The main architectural goal is to avoid god objects and keep responsibilities narrow and testable.

Folder-Based Modules in a Single App Target

Most of the app still builds as a single Xcode target organized by directory, rather than as many Swift packages. That keeps build complexity low while the architecture is still settling.

NamuKit Avoids UI Imports

Core logic lives in NamuKit and avoids SwiftUI/UIKit imports. The main exception is AppKit where Ghostty surface hosting requires it.

TerminalBackend Boundary

TerminalBackend defines the abstraction boundary, while TerminalSession is the concrete Ghostty-backed implementation. Session lifecycle is modeled explicitly through SessionState.

Immutable PaneTree

Layout state is modeled as an indirect enum. Operations return new trees instead of mutating nested nodes in place, which keeps persistence and snapshotting simple.

Focus Policy

Only workspace.select and pane.focus are allowed to steal focus. Other commands preserve the current user focus state.

Async AI by Default

NamuAI is not an agent bridge or MCP wrapper. It interprets natural language, emits structured tool calls, runs those calls through safety checks, and returns results asynchronously.

Historical Gateway Docs

Some older repo-local docs still mention a standalone gateway/WebSocket bridge. That module is not part of the active checkout. Current remote behavior is based on RelayServer plus the helper under daemon/remote/.
